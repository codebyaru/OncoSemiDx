OncoSemiDx AI Model Workflow

From Data Collection to Clinical Demonstration

1 Data Collection & Preprocessing
  • Source: Public genomic/microarray datasets (e.g., TCGA, GEO).
  • Process: Clean raw data, perform normalization (e.g., RSEM, FPKM).
  • Labeling: Annotate samples as 'Cancer' (LUAD, LUSC) vs. 'Healthy Control'.
↓
2 Feature Extraction & Selection
  • Goal: Identify significant gene expression biomarkers or mutation patterns.
  • Methods: Use techniques like PCA, t-SNE, or correlation-based feature selection to reduce dimensionality.
↓
3 AI Model Training
  • Input: Selected biomarker signals (gene expression levels).
  • Model: XGBoost (Extreme Gradient Boosting) implemented in R.
  • Output: Cancer probability score (0.0 – 1.0).
↓
4 Model Validation & Evaluation
  • Validation Method: 5-fold Cross-Validation for robust performance assessment.
  • Key Metrics: Accuracy, Sensitivity (Recall), Specificity, and AUC.
  • Result Visualization:
    • ROC Curve: To visualize trade-off and model fidelity (AUC).
    • Pie Charts: To show class distribution (e.g., LUAD vs. LUSC) in datasets.
↓
5 Integration & Simulation
  • Score Conversion: Convert AI probability (0-1) into a clear diagnostic score.
  • Simulation: Simulate the AI score as an electronic signal reading from the MOS biosensor.
  • Linkage: Connect the model output to the website dashboard (via mock API or file).
↓
6 Deployment (Technical Demo)
  • Platform: Hosted demo (e.g., using Flask, Streamlit, or Shiny for R).
  • User Interaction: Judge/User uploads a CSV of mock genomic data.
  • Final Output: The web app displays a clear, instant result.
    Example: “Cancer Risk: 82% (High)”
